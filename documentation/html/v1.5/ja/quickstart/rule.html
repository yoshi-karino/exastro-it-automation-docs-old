3.5. ルール作成とルール適用¶

監視アプリケーションからのアラート情報に対して実施する対処(アクション)の定義のことをルールと言います。
本項では、ルールの作成方法とその適用方法について説明します。

ルール作成¶

ルールの作成には、ディシジョンテーブルファイルを利用します。
ディシジョンテーブルファイルの取得は、画面上部のメニューの ルール ‣ ディシジョンテーブル から行います。
ディシジョンテーブルの作成 で作成した 新規ディシジョンテーブル のダウンロードボタン  、ディシジョンテーブルファイルをダウンロードします。
ディシジョンテーブルファイルを開き、ルールを追加します。
記載内容の詳細についてはここでは省略します。詳細は、アクション定義 を参照して下さい。
特に重要な項目としては、下記となります。
アラートメッセージ(正規表現可 一致)
監視アプリケーションから取得したアラートメッセージのマッチ条件を定義します。今回は、取得したアラート全てに対して、ITA を実行する条件となります。
アクションパラメータ情報（必須）
自動化ソフトウェアのアクションドライバとそのパラメータを指定します。
ITA_NAME
アクションドライバの作成 で作成したアクションドライバのアクションドライバ名を指定します。
CONDUCTOR_CLASS_ID
Exastro OASE から実行する ITA の ConductorクラスIDを指定します。
OPERATION_ID
Exastro OASE から実行する ITA の Conductor に紐付けるオペレーション ID を指定します。
また、本項で扱う全ての項目は下記となります。

ルールの記述が完了したら、ファイルを保存します。

新規トークンの払い出し¶

クイックスタートでは Exastro OASE で提供されている Web API を使ってアラート情報の投入を行います。本項では、Web API 利用で必要なトークンの払い出しを行います。
新規トークンの払い出しは、画面上部のメニューの ルール ‣ トークン払い出し から行います。
トークンの詳細については、トークン を参照してください。
画面上部にある、 新規トークン払い出し をクリックします。
トークン名
トークン名を入力します。
クイックスタートでは 新規トークン として登録します。
有効期限
トークンの有効期限を指定します。
クイックスタートでは、未記入(有効期限なし)として登録します
権限の設定
グループに割り当てる権限を定義します。
クイックスタートでは 権限あり に設定します。
項目の入力が完了したら、 トークン払い出し をクリックします。
トークンが画面上に表示されますが、後から確認できます。

ルールの検証¶

ルールの設定は、画面上部のメニューの ルール ‣ ルール から行います。
画面上部にある  ファイルを選択 をクリックし、 ルール作成 で作成したディシジョンテーブルファイルを選択します。
選択したファイルが正しいことを確認し、 アップロード をクリックします。
作業ステータス が ステージング適用完了 になっていることを確認し、ルールの検証のために  テストリクエスト をクリックします。

Tip

ファイルアップロード時に何らかの問題が発生してエラーとなった場合は、 ダウンロードボタン  をクリックし、ダウンロードした ZIP ファイルに梱包されているログファイルを確認して下さい。
ディシジョンテーブル名選択
テストを行う対象のディシジョンテーブルを選択します。
クイックスタートでは 新規ディシジョンテーブル を選択します。
 テストリクエスト設定へ をクリックします。
アラートメッセージ
テストリクエストで送信するメッセージを設定します。
単発テスト でアラートメッセージを This is test alert. と入力します。
 実行 をクリックします。
テストリクエストで送信したメッセージがディシジョンテーブルファイルで定義したルールにマッチしていることを確認します。
 閉じる をクリックします。
運用ステータスを 検証完了 にするかどうかの確認があるので、OK をクリックします。

ルールの本番適用¶

ルールの本番適用は、画面上部のメニューの ルール ‣ ルール から行います。
画面上部の ステージング適用ルール にある適用ボタン  をクリックします。
数秒～数分後に、 作業ステータス が プロダクション適用完了 になります。
Exastro OASE が監視アプリケーションからアラートメッセージを取得すると、ITA の Conductor を実行します。
